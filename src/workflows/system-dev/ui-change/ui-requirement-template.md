# UI Change Requirement

## Goal

画面上で何を変えたいか。

## User Context

誰が、どの画面で、何をするための変更か。

## Target Screens / Components

- 画面
- component
- route
- 状態

## Desired Behavior

表示、操作、状態遷移、エラー/空/loading状態。

## Visual / UX Constraints

- 既存デザインシステム
- レスポンシブ要件
- アクセシビリティ要件
- 触らないUI

## Out of Scope

今回やらない画面や挙動。

## Acceptance Criteria

- 指定画面/状態で期待どおり表示される
- 操作が期待どおり動く
- 主要viewportで破綻しない

## Validation

- 実行してほしい検証コマンド
- スクリーンショット確認や手動確認手順

## Notes

参考デザイン、スクリーンショット、関連issue、関連ファイルなど。
